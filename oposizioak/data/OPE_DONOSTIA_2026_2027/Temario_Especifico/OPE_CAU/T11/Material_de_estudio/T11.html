1. La familia Windows de sistemas operativos de usuario

Windows 10, publicado en 2015, se distribuyó durante años bajo el modelo «Windows as a Service»: en lugar de nuevas versiones mayores cada pocos años, recibía actualizaciones de funciones periódicas identificadas por año y semestre (por ejemplo, 21H1, 22H2). Su última versión de funciones fue la 22H2. Windows 10 finalizó su soporte el 14 de octubre de 2025: a partir de esa fecha ya no recibe actualizaciones de seguridad gratuitas, salvo que el equipo esté inscrito en el programa ESU (Extended Security Updates), que amplía la disponibilidad de parches de seguridad críticos hasta octubre de 2027.

Windows 11, publicado en 2021, es la versión actual de la familia. Mantiene el mismo núcleo NT (Tema 9) y gran parte de la compatibilidad de aplicaciones de Windows 10, pero introduce una interfaz rediseñada (menú Inicio centrado, Fluent Design) y, sobre todo, unos requisitos de hardware más estrictos, orientados a reforzar la seguridad del equipo desde el propio firmware. Se distribuye igualmente mediante actualizaciones de funciones periódicas (las versiones más recientes en el momento de escribir este tema son 24H2 y 25H2).

Ediciones

  • Home: orientada al uso doméstico; no admite unirse a un dominio de Directorio Activo.
  • Pro: añade funciones orientadas a empresa y profesionales autónomos: unión a dominio, BitLocker, Escritorio remoto como servidor, Hyper-V cliente.
  • Enterprise y Education: ediciones de volumen para grandes organizaciones y centros educativos, con funciones adicionales de seguridad y gestión centralizada.
  • Enterprise LTSC (Long-Term Servicing Channel): canal especial, sin las actualizaciones de funciones periódicas del resto de ediciones, pensado para equipos críticos que necesitan máxima estabilidad (cajeros, equipos industriales) a cambio de un ciclo de vida mucho más largo entre versiones mayores.
📌 Nivel básico — imprescindible para el examen
Windows 10 ya ha llegado al final de su ciclo de soporte estándar (14/10/2025). Un CAU debe tener presente este dato porque condiciona directamente su trabajo diario: los equipos que aún ejecuten Windows 10 sin estar acogidos al programa ESU dejan de recibir parches de seguridad, lo que los convierte en un riesgo y suele obligar a planificar su migración a Windows 11.

2. Características técnicas y funcionalidades

Windows 10 y 11 se basan en el núcleo NT, de arquitectura híbrida (Tema 9), con soporte nativo de 64 bits (x64) como estándar actual —la arquitectura de 32 bits (x86) ha quedado relegada a equipos muy antiguos— y un soporte creciente de la arquitectura ARM64 en portátiles orientados a la eficiencia energética.

Requisitos y funciones de seguridad de Windows 11

  • TPM 2.0 (Trusted Platform Module): chip dedicado (físico o firmware) que almacena de forma segura claves criptográficas, usado por funciones como BitLocker o Windows Hello.
  • Secure Boot (arranque seguro): mecanismo de UEFI que verifica la firma digital del cargador de arranque y de los componentes que se cargan antes que el propio sistema operativo, para impedir que software malicioso de bajo nivel (rootkits, bootkits) se ejecute antes de que Windows tome el control.
  • Procesador compatible: Windows 11 exige un procesador de una lista concreta de modelos soportados, más recientes que los admitidos por Windows 10.

Funcionalidades destacadas

  • Windows Update: mecanismo integrado de descarga e instalación de actualizaciones de seguridad y de funciones.
  • Microsoft Defender Antivirus: antivirus y protección contra amenazas integrado de serie en el sistema (se amplía en el Tema 25).
  • BitLocker: cifrado de disco completo, apoyado en el TPM.
  • Hyper-V (cliente): hipervisor de tipo 1 (Tema 10) incluido en las ediciones Pro y superiores, útil para máquinas virtuales de prueba en el propio equipo de trabajo.
  • WSL (Windows Subsystem for Linux): permite ejecutar un entorno Linux integrado dentro de Windows, sin necesidad de una máquina virtual completa.
  • Windows Hello: autenticación biométrica (huella, reconocimiento facial) o mediante PIN, ligada al dispositivo.

La interfaz de configuración también evoluciona: la aplicación moderna Configuración sustituye progresivamente al Panel de control clásico, aunque este último se mantiene disponible para algunas opciones avanzadas.

3. Servicios y recursos del sistema

  • Servicios de Windows: procesos en segundo plano que se ejecutan sin interacción directa del usuario (por ejemplo, el servicio de impresión o el de Windows Update). Se administran desde la consola de servicios (services.msc), donde cada servicio puede configurarse como automático, automático (inicio retrasado), manual o deshabilitado.
  • Administrador de tareas (Task Manager): herramienta para supervisar procesos en ejecución, rendimiento de CPU/memoria/disco/red, aplicaciones que arrancan con el sistema, usuarios conectados y el propio historial de uso de aplicaciones.
  • Registro de Windows (Windows Registry): base de datos jerárquica central donde se almacena la configuración del sistema operativo y de las aplicaciones instaladas, organizada en grandes ramas o «hives» (HKEY_LOCAL_MACHINE, HKEY_CURRENT_USER, HKEY_CLASSES_ROOT, HKEY_USERS, HKEY_CURRENT_CONFIG). Se edita con la herramienta regedit; una modificación incorrecta puede dejar el sistema inestable o inarrancable, por lo que conviene exportar una copia antes de editarlo.
  • Visor de eventos (Event Viewer): recopila los registros (logs) de sucesos del sistema, la seguridad y las aplicaciones. Su uso y supervisión práctica se desarrolla con detalle en el Tema 15.
  • MMC (Microsoft Management Console): marco común sobre el que se construyen la mayoría de consolas de administración de Windows, mediante módulos denominados snap-ins (por ejemplo, la propia consola de servicios o la de usuarios y grupos locales).
  • PowerShell: intérprete de comandos y lenguaje de script orientado a objetos (no a texto plano, como el símbolo del sistema clásico, CMD), que se ha convertido en la herramienta de referencia para la administración avanzada y la automatización en Windows.
⚠️ No confundir
No debe confundirse el Registro de Windows (la base de datos de configuración del sistema) con el Visor de eventos (el histórico de sucesos que han ocurrido). Ambos son fuentes de información para el diagnóstico de incidencias, pero cumplen funciones completamente distintas y se consultan con herramientas diferentes (regedit frente al propio Visor de eventos).

4. Instalación

Métodos de instalación

  • Medio físico de arranque: un USB o DVD de instalación, creado habitualmente con la herramienta oficial Media Creation Tool, desde el que se arranca el equipo para instalar el sistema de forma manual.
  • Instalación en red: en entornos corporativos, mediante arranque por red (PXE Boot) mediante servicios como WDS (Windows Deployment Services), que permite instalar el sistema operativo en varios equipos sin necesidad de medios físicos individuales.
  • Despliegue mediante imágenes personalizadas: se prepara un equipo de referencia con el sistema, las aplicaciones y la configuración deseada, se generaliza con Sysprep (elimina la información específica del equipo, como el identificador de seguridad, para que la imagen pueda clonarse) y se captura como imagen en formato WIM (Windows Imaging Format), gestionable con la herramienta DISM (Deployment Image Servicing and Management). Esa imagen puede desplegarse después en muchos equipos de forma idéntica.

Tipos de instalación

  • Instalación limpia (clean install): se instala el sistema operativo sobre un disco vacío o formateado, sin conservar aplicaciones ni configuración previas.
  • Actualización en el sitio (in-place upgrade): se instala la nueva versión sobre la instalación existente, conservando aplicaciones, configuración y archivos del usuario.

Activación

En entornos domésticos, la activación se realiza con una clave de producto individual. En entornos corporativos, es habitual la activación por volumen, mediante KMS (Key Management Service), que activa periódicamente los equipos de la organización contra un servidor interno, o MAK (Multiple Activation Key), una clave única que permite activar un número determinado de equipos directamente contra los servidores de Microsoft.

📌 Nivel básico — imprescindible para el examen
DISM y Sysprep son herramientas distintas y complementarias en el despliegue masivo: Sysprep generaliza (limpia la identidad) de un equipo de referencia antes de capturarlo como imagen; DISM se usa para servir, montar, modificar e instalar esas imágenes WIM (por ejemplo, para inyectar controladores o actualizaciones sin necesidad de volver a instalar el sistema desde cero).

5. Administración y mantenimiento

Actualizaciones

Windows Update gestiona la descarga e instalación de actualizaciones en el equipo individual. En entornos gestionados, esa distribución puede centralizarse: WSUS (Windows Server Update Services) permite descargar las actualizaciones una sola vez en un servidor interno y distribuirlas de forma controlada a los equipos de la organización. Microsoft anunció en 2024 la deprecación de WSUS: sigue siendo totalmente funcional y soportado (incluido en Windows Server 2025), pero no recibirá nuevas funciones, y la orientación de Microsoft es migrar la gestión de actualizaciones de cliente hacia soluciones en la nube, como Windows Update for Business o Windows Autopatch, integradas con Microsoft Intune.

Administración remota

  • Escritorio remoto (RDP, Remote Desktop Protocol): acceso gráfico remoto al equipo, disponible como servidor en la edición Pro y superiores.
  • PowerShell remoto: ejecución de comandos y scripts de administración sobre uno o varios equipos remotos.
  • Directiva de grupo (GPO, Group Policy Object): mecanismo de configuración centralizada de equipos unidos a un dominio de Directorio Activo (se estudia junto al propio Directorio Activo en el Tema 12).
  • Microsoft Intune: plataforma de gestión unificada de endpoints (MDM, Mobile Device Management) en la nube, alternativa/complemento moderno a la Directiva de grupo tradicional, especialmente relevante para equipos que no están permanentemente conectados a la red corporativa.

Mantenimiento habitual

  • Liberador de espacio en disco: elimina archivos temporales, copias de actualizaciones antiguas y otros datos prescindibles.
  • Desfragmentación: reorganiza los datos de un disco HDD (Tema 6) para reducir la fragmentación y mejorar el tiempo de acceso; no es aplicable ni recomendable en discos SSD, donde en su lugar el sistema ejecuta periódicamente la operación TRIM, que informa al SSD de qué bloques ya no contienen datos válidos.
  • Comprobación de errores de disco (chkdsk): analiza y repara errores en el sistema de archivos y en los sectores del disco.
  • Monitor de rendimiento y fiabilidad: registra el histórico de estabilidad del equipo (bloqueos, errores, instalaciones) y el rendimiento en tiempo real de sus componentes.

6. Recuperación del sistema

  • Puntos de restauración (System Restore): capturas del estado de la configuración y del Registro del sistema en un momento dado, que permiten deshacer cambios recientes (por ejemplo, tras instalar un controlador problemático) sin afectar a los archivos personales del usuario.
  • Copia de seguridad: el Historial de archivos protege carpetas de usuario concretas con copias incrementales, mientras que una imagen del sistema captura el disco completo, incluyendo el sistema operativo, para restaurarlo íntegramente en caso de fallo grave.
  • WinRE (Windows Recovery Environment): entorno de recuperación accesible al arrancar el equipo, que ofrece reparación de inicio automática, restauración desde una imagen del sistema, restauración desde un punto de restauración, acceso a un símbolo del sistema de recuperación y la opción de reiniciar el equipo.
  • Medios de recuperación: una unidad USB de recuperación, creada desde el propio sistema mientras funciona correctamente, permite acceder a estas mismas opciones aunque el equipo no llegue a arrancar.
⚠️ No confundir
No deben confundirse «Restaurar sistema» y «Reiniciar este PC» (o «Reset»). Restaurar sistema únicamente revierte la configuración y el Registro a un punto anterior, sin tocar los documentos personales del usuario. Reiniciar este PC reinstala el sistema operativo, con la opción de conservar los archivos personales o eliminarlos por completo junto con las aplicaciones instaladas; es una operación mucho más drástica, pensada para cuando el sistema está gravemente dañado o se va a entregar el equipo a otra persona.

7. Gestión del software base: instalación y desinstalación de productos

Formatos y métodos de instalación

  • Instaladores tradicionales (.exe): ejecutables generados con herramientas como InstallShield, NSIS o Inno Setup; cada uno define su propia lógica de instalación.
  • MSI (Windows Installer): formato estándar de Windows para la instalación de software, basado en una base de datos de componentes; permite instalaciones silenciosas (sin intervención del usuario) y su distribución centralizada mediante GPO o herramientas de gestión de software.
  • APPX / MSIX: formatos modernos de aplicaciones empaquetadas, propios de la Microsoft Store, que aíslan mejor la aplicación del resto del sistema y facilitan una desinstalación limpia.
  • Gestores de paquetes en línea de comandos: winget (Windows Package Manager, oficial de Microsoft) permite instalar, actualizar y desinstalar aplicaciones desde la terminal, de forma similar a como funcionan los gestores de paquetes en Linux (Tema 13); Chocolatey es una alternativa de la comunidad con un enfoque similar.

Desinstalación

El software instalado se desinstala desde la aplicación Configuración > Aplicaciones, o desde el clásico Panel de control > Programas y características. Los paquetes MSI también pueden desinstalarse desde línea de comandos con `msiexec /x`. Una desinstalación correcta debe eliminar, además del propio programa, los componentes y entradas de Registro asociados, para evitar dejar residuos que puedan interferir con instalaciones futuras.

🔍 Para ampliar — contexto útil, menor probabilidad de pregunta directa
La distribución de software a gran escala en un entorno con muchos equipos —con qué herramientas se empaqueta, distribuye y mantiene actualizado el software de forma centralizada— se desarrolla en profundidad en el Tema 16, dedicado a herramientas ofimáticas y distribución de software en entornos distribuidos; aquí basta con conocer los formatos y mecanismos básicos de instalación y desinstalación a nivel de un único equipo.

8. Resumen esquemático

  • Familia Windows: Windows 10 (fin de soporte 14/10/2025, ESU hasta 10/2027) y Windows 11 (versión actual, mayores requisitos de seguridad: TPM 2.0, Secure Boot); ediciones Home, Pro, Enterprise/Education y Enterprise LTSC.
  • Características técnicas: núcleo NT híbrido, 64 bits estándar; funciones clave: Windows Update, Microsoft Defender, BitLocker, Hyper-V cliente, WSL, Windows Hello.
  • Servicios y recursos: servicios de Windows (services.msc), Administrador de tareas, Registro (regedit), Visor de eventos (detalle en el Tema 15), MMC y PowerShell.
  • Instalación: medio físico, red (PXE/WDS) o imágenes personalizadas (Sysprep + DISM sobre formato WIM); instalación limpia frente a actualización en el sitio; activación individual, KMS o MAK.
  • Administración y mantenimiento: Windows Update/WSUS (deprecado desde 2024, alternativa en la nube: Windows Autopatch/Intune) o GPO/Intune para gestión centralizada; liberador de espacio, TRIM (SSD) frente a desfragmentación (HDD), chkdsk.
  • Recuperación: puntos de restauración (solo configuración), copias de seguridad, WinRE y medios de recuperación; «Restaurar sistema» frente a «Reiniciar este PC».
  • Software base: formatos .exe, MSI, APPX/MSIX; gestores de paquetes winget/Chocolatey; desinstalación desde Configuración, Panel de control o msiexec /x.
