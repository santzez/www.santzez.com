1. Sistemas operativos de servidor: la familia Windows Server

Un sistema operativo de servidor está optimizado para permanecer en funcionamiento de forma continua (24/7), atender muchas conexiones simultáneas y administrar recursos compartidos de red, frente a un sistema operativo de usuario (Tema 11), pensado para el uso individual de un único puesto de trabajo. Windows Server admite además una modalidad de instalación mínima, Server Core, sin entorno gráfico de escritorio, administrable por PowerShell o de forma remota, que reduce la superficie de ataque y la necesidad de reinicios frente a la instalación con Desktop Experience (interfaz gráfica completa).

La versión más reciente de la familia es Windows Server 2025 (disponible desde finales de 2024), que convive todavía con un parque muy amplio de servidores en Windows Server 2022 y versiones anteriores. Entre las novedades más relevantes de Windows Server 2025 destacan mejoras de fondo en Directorio Activo (punto 4), soporte de NUMA para más de 64 núcleos por servidor, cifrado y firma LDAP obligatorios por defecto, TLS 1.3 como protocolo por defecto, y —en su edición Azure— la posibilidad de aplicar parches de seguridad sin reiniciar el servidor (hotpatching).

Ediciones principales

Edición Virtualización incluida Uso típico
Standard Hasta 2 máquinas virtuales por licencia Entornos físicos o poco virtualizados
Datacenter Virtualización ilimitada Centros de datos muy virtualizados, funciones avanzadas de almacenamiento y clúster

2. Instalación y configuración: roles y características

La instalación de Windows Server admite los mismos métodos vistos para Windows cliente en el Tema 11 (medio físico, despliegue por red, imágenes personalizadas con Sysprep y DISM), eligiendo además entre instalación Server Core o con Desktop Experience.

Una vez instalado, el sistema base se especializa añadiendo roles y características desde el Administrador del servidor (Server Manager) o mediante PowerShell (cmdlet `Install-WindowsFeature`), y estas mismas herramientas permiten gestionar de forma centralizada varios servidores a la vez desde un único punto.

  • Rol (server role): función principal que el servidor presta a la red. Ejemplos: AD DS (Servicios de dominio de Directorio Activo), servidor DNS, servidor DHCP, Servicios de archivos y almacenamiento, servidor web (IIS), Hyper-V, Servicios de escritorio remoto.
  • Característica (feature): componente de soporte que complementa al sistema, sin constituir por sí mismo un servicio de red independiente. Ejemplos: herramientas de copia de seguridad de Windows Server, Failover Clustering (clúster de conmutación por error), las herramientas de administración remota del servidor (RSAT).
⚠️ No confundir
Un rol y una característica no son lo mismo, y el examen puede pedir distinguirlos: el rol define qué servicio presta el servidor hacia la red (por ejemplo, ser controlador de dominio o servidor DNS); la característica es un componente auxiliar que da soporte al sistema o a otros roles, pero no ofrece por sí solo un servicio de red identificable.

3. Mantenimiento básico: actualizaciones y monitorización

Actualizaciones

Microsoft publica sus actualizaciones de seguridad de forma mensual, el segundo martes de cada mes («Patch Tuesday»). En un entorno de servidores, la buena práctica habitual es no instalar los parches directamente en producción: se prueban antes en un entorno de preproducción y se aplican en una ventana de mantenimiento planificada, con una copia de seguridad o una instantánea (snapshot, Tema 10) previa por si es necesario revertir el cambio.

Para la distribución centralizada de actualizaciones, WSUS (Windows Server Update Services) sigue disponible como rol en Windows Server 2025, aunque Microsoft lo declaró en deprecación desde 2024: continúa soportado, pero sin nuevas funciones, y la orientación de Microsoft es ir migrando la gestión de parches hacia herramientas en la nube, como Azure Update Manager.

Monitorización

  • Administrador de tareas y Monitor de rendimiento: igual que en el equipo de usuario (Tema 11), permiten supervisar el consumo de CPU, memoria, disco y red en tiempo real.
  • Visor de eventos: recopila los registros de sistema, seguridad y aplicaciones también en el servidor; su manejo detallado se estudia en el Tema 15.
  • Monitor de recursos (Resource Monitor): vista más detallada del uso de recursos por proceso, útil para diagnosticar cuellos de botella.
  • Windows Admin Center: consola de administración moderna, basada en navegador web, que centraliza la gestión de varios servidores (Server Core incluido) sustituyendo progresivamente a distintas consolas MMC clásicas.
  • System Center Operations Manager (SCOM): solución de monitorización a gran escala para entornos empresariales con muchos servidores, con alertado proactivo ante problemas de rendimiento o disponibilidad.

4. Fundamentos de Directorio Activo

Directorio Activo (Active Directory, AD) es el servicio de directorio de Microsoft: una base de datos centralizada y jerárquica que almacena información sobre los objetos de una red —usuarios, equipos, grupos, impresoras compartidas— y ofrece los servicios de autenticación y autorización necesarios para que esos objetos puedan identificarse y acceder a los recursos de la red de forma centralizada y segura. El servicio concreto que implementa el Directorio Activo en Windows Server se llama AD DS (Active Directory Domain Services).

AD DS se apoya en dos protocolos estándar: LDAP (Lightweight Directory Access Protocol), para consultar y modificar los objetos del directorio, y Kerberos, para la autenticación segura de usuarios y equipos.

📌 Nivel básico — imprescindible para el examen
Un Directorio Activo funciona como una gran guía y organigrama centralizado de toda la red: en lugar de que cada equipo mantenga su propia lista de usuarios y contraseñas, existe un único punto central que sabe quién es cada usuario, a qué grupos pertenece, qué equipos existen y qué permisos tiene cada cual sobre los recursos compartidos.
🔍 Para ampliar — contexto útil, menor probabilidad de pregunta directa
No debe confundirse AD DS con Microsoft Entra ID (el nuevo nombre, desde 2023-2024, de Azure Active Directory): Entra ID es un servicio de identidad en la nube, orientado a aplicaciones y servicios en línea, que no organiza los objetos en dominios, árboles y bosques como AD DS. Es habitual un escenario híbrido, en el que las cuentas de un AD DS local se sincronizan con Entra ID (mediante Entra Connect) para que los usuarios puedan acceder tanto a recursos locales como a servicios en la nube con una única identidad. El temario de este tema se centra en los fundamentos del Directorio Activo «clásico» (AD DS), no en Entra ID.

5. Estructura lógica y física del Directorio Activo

Estructura lógica

  • Dominio: unidad administrativa básica de AD; un conjunto de objetos (usuarios, equipos, grupos) que comparten una misma base de datos de directorio y unas mismas políticas de seguridad.
  • Árbol de dominios: conjunto de uno o más dominios que comparten un espacio de nombres DNS contiguo (por ejemplo, `donostiatik.local` y `sedes.donostiatik.local`).
  • Bosque (forest): conjunto de uno o más árboles de dominios; no exige un espacio de nombres común, pero sí comparte un mismo esquema y catálogo global. Es el límite máximo de seguridad y de administración dentro de un despliegue de AD.
  • Unidad organizativa (OU, Organizational Unit): contenedor dentro de un dominio que permite organizar los objetos de forma jerárquica (por departamento, ubicación, función) y aplicarles Directivas de grupo (GPO) de forma selectiva.
  • Relación de confianza (trust): vínculo que permite que los usuarios de un dominio o bosque accedan a recursos de otro dominio o bosque distinto.

Estructura física

  • Sitio (site): agrupación de una o varias subredes IP bien conectadas entre sí (habitualmente correspondiente a una ubicación física, como un edificio o un centro de datos), que AD utiliza para optimizar el tráfico de replicación entre controladores de dominio y para que un usuario inicie sesión contra el controlador de dominio más cercano en vez de contra uno remoto.
  • Controlador de dominio (Domain Controller, DC): servidor que aloja una copia de la base de datos de AD de su dominio y presta los servicios de autenticación y consulta del directorio.
⚠️ No confundir
Dominio y sitio son conceptos independientes y no deben confundirse: el dominio es una agrupación lógica de seguridad y administración; el sitio es una agrupación física basada en la conectividad de red. Un mismo dominio puede extenderse por varios sitios (por ejemplo, varias sedes de un organismo), y un mismo sitio puede alojar controladores de dominio de más de un dominio del bosque.

6. Componentes técnicos: controladores de dominio y roles FSMO

Para dar continuidad de servicio, un dominio suele contar con varios controladores de dominio, que se replican entre sí. AD funciona de forma multimaestro para la mayoría de las operaciones: un cambio puede hacerse en cualquier controlador de dominio y se replica hacia el resto.

  • Catálogo global (Global Catalog): copia parcial —con los atributos más consultados— de todos los objetos de todos los dominios del bosque, alojada en determinados controladores de dominio, para acelerar las búsquedas que abarcan todo el bosque.
  • Esquema (schema): definición de qué clases de objetos (usuario, equipo, grupo…) y qué atributos pueden existir en el directorio; es común a todo el bosque.
  • GPO (Group Policy Object, Directiva de grupo): conjunto de configuraciones que se vincula a un dominio, un sitio o una OU, y se aplica automáticamente a los usuarios y equipos incluidos en su ámbito, tal y como se introdujo en el Tema 11.
  • LAPS (Local Administrator Password Solution): solución, reforzada en Windows Server 2025, para gestionar de forma centralizada y con rotación periódica la contraseña de la cuenta de administrador local de cada equipo unido al dominio.

Aunque AD DS es multimaestro para la mayoría de cambios, existen cinco operaciones que, para evitar conflictos, solo puede realizar un único controlador de dominio a la vez. Son los roles FSMO (Flexible Single Master Operations): dos a nivel de bosque y tres a nivel de dominio.

Rol FSMO Ámbito Función
Maestro de esquema (Schema Master) Bosque Único controlador que puede modificar el esquema del directorio
Maestro de nomenclatura de dominios (Domain Naming Master) Bosque Controla la adición y eliminación de dominios en el bosque
Maestro de RID (RID Master) Dominio Asigna bloques de identificadores relativos (RID) para crear nuevos objetos con SID único
Emulador de PDC (PDC Emulator) Dominio Referencia horaria y compatibilidad con versiones y clientes antiguos; procesa cambios de contraseña con prioridad
Maestro de infraestructura (Infrastructure Master) Dominio Mantiene actualizadas las referencias a objetos de otros dominios (por ejemplo, en pertenencia a grupos)
📌 Nivel básico — imprescindible para el examen
Para recordar el reparto de los roles FSMO: el bosque tiene dos roles únicos (Esquema y Nomenclatura de dominios), y cada dominio del bosque tiene sus propios tres roles (RID, Emulador de PDC e Infraestructura). Si un dominio de AD se apoya también en DNS —y siempre lo hace, porque AD DS depende de DNS para que los equipos localicen a los controladores de dominio— lo habitual es que el propio controlador de dominio actúe también como servidor DNS, con zonas integradas en el directorio.

7. Resumen esquemático

  • Windows Server: sistema operativo de servidor, con instalación Server Core (sin GUI) o Desktop Experience; versión actual Windows Server 2025, muy extendido aún Windows Server 2022; ediciones Standard (virtualización limitada) y Datacenter (ilimitada).
  • Roles y características: el rol presta un servicio de red (AD DS, DNS, DHCP, IIS…); la característica da soporte auxiliar (backup, clustering, RSAT); se gestionan desde el Administrador del servidor o PowerShell.
  • Actualizaciones: Patch Tuesday mensual, prueba en preproducción antes de producción; WSUS deprecado desde 2024 (sigue soportado) frente a Azure Update Manager como alternativa en la nube.
  • Monitorización: Administrador de tareas, Monitor de rendimiento y de recursos, Visor de eventos (Tema 15), Windows Admin Center y SCOM para entornos grandes.
  • Directorio Activo (AD DS): servicio de directorio centralizado sobre LDAP y Kerberos; distinto de Microsoft Entra ID (identidad en la nube), aunque pueden convivir en modo híbrido.
  • Estructura lógica: dominio, árbol de dominios, bosque (límite de seguridad), unidad organizativa (OU) y relaciones de confianza.
  • Estructura física: sitios (subredes bien conectadas) y controladores de dominio (DC).
  • Componentes técnicos: catálogo global, esquema, GPO, LAPS y los cinco roles FSMO (2 de bosque: Esquema y Nomenclatura de dominios; 3 de dominio: RID, PDC Emulator e Infraestructura).
